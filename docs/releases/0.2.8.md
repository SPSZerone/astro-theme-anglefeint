# Release Notes: 0.2.8

## Highlights

This release aligns the published theme package with **stable Astro 6.x** peer expectations and brings the maintainer workspace onto **Astro 6.1.3** with current `@astrojs/*` integration versions.

## Major Themes

### Package peer contract

- `peerDependencies.astro` is now `^5.0.0 || ^6.0.0` (replacing the prior beta-oriented upper bound) so consumers on Astro 6 stable no longer depend on a prerelease peer range.

### Workspace / template baseline

- Root `astro` and official integrations (`@astrojs/mdx`, `@astrojs/rss`, `@astrojs/sitemap`, `@astrojs/check`) are upgraded to the stable 6.1 line used by `npx @astrojs/upgrade`.
- User-facing README badges and stack notes were updated to match the stable baseline.

## Migration Notes

- **From npm:** run `npm update @anglefeint/astro-theme` (or pin `0.2.8`), then ensure your app uses **Astro 6.1.x** (or compatible 6.x) and matching `@astrojs/*` versions; follow [Upgrade Astro](https://docs.astro.build/en/upgrade-astro/) if you lag behind.
- **Peer warnings:** if you still pin Astro 5, peers remain satisfied via `^5.0.0`.
- **Content / config:** no `src/site.config.ts` or content schema changes are required for this release.

## Starter Sync

- Starter sync is required for this release so `starter` picks up the new theme version range and lockfile alignment.

## Validation Summary

- `npm run check:docs`
- `npm run maintainer:sync-starter:check`
- `npm run check`
